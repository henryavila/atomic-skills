## Summary
The macOS timeout fallback documented for Codex invocation is not executable as written, so the portability fix fails on the default macOS path where `timeout`/`gtimeout` are absent and `perl` is present. There are also two edge-case regressions in newly changed behavior: fenced-code detection still misclassifies mixed-fence Markdown, and the installer points users at a cleanup override flag that the CLI does not implement.

## Findings

### F-001 [major] correctness — skills/shared/codex-bridge-assets/invocation-canonical.txt:23-40

**Evidence:**
```bash
if command -v timeout >/dev/null 2>&1; then
  TIMEOUT_CMD="timeout"
elif command -v gtimeout >/dev/null 2>&1; then
  TIMEOUT_CMD="gtimeout"           # macOS via `brew install coreutils`
elif command -v perl >/dev/null 2>&1; then
  TIMEOUT_CMD="perl -e 'alarm shift @ARGV; exec @ARGV' --"   # POSIX-portable fallback
else
  echo "ERROR: no timeout wrapper available. Install coreutils (Linux: apt/yum; macOS: brew install coreutils) or perl." >&2
  exit 1
fi
```

**Claim:** The Perl fallback stored in `TIMEOUT_CMD` is split by the shell without re-parsing embedded quotes, so `$TIMEOUT_CMD <SECONDS> codex ...` passes a broken `-e` program to Perl.

**Impact:** On default macOS machines without GNU coreutils, Codex review invocation fails instead of timing out portably; users hit a shell/Perl error before `codex` runs.

**Recommendation:** Replace the string command with a shell function or explicit branch invocation, e.g. call `perl -e 'alarm shift @ARGV; exec @ARGV' -- "$seconds" codex ...` directly in the Perl branch.

**Confidence:** high

---

### F-002 [minor] correctness — src/bootstrap.js:306-311

**Claim:** `detectPlanShape` toggles fence state on either ``` or `~~~`, so a `~~~` line inside a backtick fence, or a ``` line inside a tilde fence, incorrectly closes the fence.

**Impact:** Markdown sources containing nested or mixed fence examples can still count code-block phase headings as real headings, causing `project-plan discover` to misclassify an initiative as a multi-phase plan.

**Recommendation:** Track the opening fence marker character and length, and only close on a matching fence marker that satisfies Markdown closing rules.

**Confidence:** high

---

### F-003 [minor] correctness — src/install.js:571-576

**Claim:** The installer tells users to pass `--force-legacy-cleanup`, but the CLI option is not implemented.

**Impact:** Users following the non-interactive cleanup instructions get an unknown-option failure and still cannot override preservation from the documented path.

**Recommendation:** Either implement and test `--force-legacy-cleanup`, or remove the flag suggestion and instruct manual removal only.

**Confidence:** high
