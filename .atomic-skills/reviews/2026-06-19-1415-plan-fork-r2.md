## Summary
The plan leaves core state contracts undefined while multiple later phases depend on those contracts for validation, cycle detection, focus resolution, archive propagation, and cross-worktree writes. It also orders parallel-mode behavior before the canonical parent-state protocol exists, which makes the planned cycle-check/write path unstable.

Several gates can pass without proving their stated outcomes: conflict handling is underspecified, resume behavior relies on an undefined interactive offer, and the aiDeck handoff verifier checks only a subset of the required documentation.

**Recommendation:** Add an F0 task that defines the complete schema contract for both fields, including examples and negative cases, then require `validate-state` tests to validate that contract before F1 starts.

## Summary

They also expose a stricter compatibility failure: the plan treats `plan.md`/phase frontmatter as both canonical storage for new fields and aiDeck-facing strict state, while the only aiDeck work is deferred to a final documentation handoff.
