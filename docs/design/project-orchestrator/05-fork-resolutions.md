## Cluster A — Mode 2 execution & cross-agent portability

### F-A1 — Mode 2 v1 merge-back strategy
**Decision:** Keep worktree isolation; **merge-back is operator-prompted** (the skill stages the exact `git -C <primary> merge --no-ff <wt-ref>` command + conflict-abort instruction + worktree path, then STOPS the done-transition, task stays `active`), and after the operator confirms the merge the skill **MANDATORILY re-runs the verifier against the primary tree** before writing `status:done` + evidence. Deterministic serial-rebase automation = the named v2 follow-on; do not block v1 on it.
**Why:** Only this option simultaneously preserves single-threaded-coding (R-EXEC-13, exactly one gated writer on primary), keeps the worktree as a real blast-radius firewall, and keeps the post-merge re-verify skill-owned (the non-fakeable catch for "both tasks pass in-worktree but conflict/regress on merge"). v1 is contractually barred from claiming the unattended use case, so building serial-rebase now buys a capability v1 can't advertise — pure cost on the highest-integrity surface. Shipping without isolation (option b) puts a foreign writing model on the tree the operator is editing — the single largest integrity hazard, and it violates the "codex write-mode fenced out of the live tree" dogfood rule.
**Falsifier:** A 2-task codex batch editing adjacent lines of one file, each green in its own worktree, that ever marks BOTH `done` while the merged primary fails the verifier (the mandatory post-merge re-verify leaked).
**Binds:** Inc5. Cross-deps: F-A3, F-A6.

### F-A5 — `minBatchTasks` K floor  *(under-justification caveat folded in)*
**Decision:** **K = 4** independent dispatch-eligible tasks (config `mode2.minBatchTasks`, default 4), **explicitly labeled interpolated-pending-data** rather than derived. Firing rule stays `N≥K ∧ ≥2 pairwise-independent ∧ not cohesive`. v2 replaces the fixed K with a token-weighted threshold once `dispatch-log.json` carries real per-task overhead.
> **Caveat (consistency critic):** 4 is bracketed between two soft anchors (parallel-dispatch's ~3 floor + the verdict's hedged ~5); the "past the 1-2 theater zone" argument only justifies K>2. Treat 4 as a defensible v1 placeholder to be validated against data, not a derived constant.

**Falsifier:** After ≥3 real batches with the sidecar populated, N=4 batches systematically cost more end-to-end than the Mode-1 counterfactual (K too low), or N=3 would have consistently won (K too high).
**Binds:** Inc5. Cross-deps: F-A6, F-A7.

### F-A7 — Cohesion classification (F1 gate)
**Decision:** **User-confirmed, evidence-assisted, two-stage.** The agent first auto-runs the deterministic pairwise-grep disjointness check (reuse parallel-dispatch Phase 1.3/1.5 → grep-output evidence), then presents that evidence and asks the user the single binary F1 question. The user's answer is the gate value; **disagreement (grep clean but user says cohesive, or vice-versa) HALTs to Mode 1** (fail-closed). The agent never silently classifies cohesive-as-independent.
**Why:** F1 is the hard disqualifier and highest-stakes misclassification — agent-inference self-grades the single most consequential gate question (violates anti-vanity #6) and the agent has a standing incentive to answer "independent." Grep proves path-disjointness (necessary) but not semantic cohesion (two disjoint-file tasks can be one coordinated contract change), so the human must adjudicate the semantic part; the grep evidence makes that question answerable rather than blind.
**Falsifier:** A batch with disjoint file sets (clean grep) but one semantically-cohesive change (A defines an API contract, B/C consume A's exact signature) that the gate marks F1=YES on the clean grep alone, without asking the user.
**Binds:** Inc5. Cross-deps: F-A5.

**Decision — two layers, no hard size-block:**
- **Layer 1 (hard floor, KEEP):** the existing mandatory local `review-code` at every `phase-done` (project-transitions.md:93) — magnitude-independent, fires every phase close unless `--skip-review` + a verbatim recorded+surfaced reason.
- **Layer 2 (codex escalation):** the conditional cross-model/codex review, **magnitude-laddered**, reusing the drift thresholds in project-drift.md:115-116. **Yellow ("recommended", advisory — surfaced, not auto-run)** = 1-3 commits or 1-100 lines un-reviewed since `lastReviewedCommit`; **Red ("overdue", MANDATORY — auto-runs, see escalation rule below)** = ≥4 commits **or** ≥100 lines un-reviewed **or** ≥7 days **or** a `detect-scope`-ratified scope-drift item touching the phase **or** a phase-done since the last codex review. **No numeric threshold hard-BLOCKS phase advance** — magnitude changes whether the codex review is *recommended* (yellow) vs *runs automatically* (red), never whether the phase can close.
> **Caveat (consistency critic):** the first pass tightened the red line 100→80; that knob is advisory/non-load-bearing by the decision's own admission, so the tightening carries no falsifiable consequence. **Keep the existing 100-line threshold** — don't change a number just to change it. The genuinely-new, non-gameable signal is the **scope-drift trigger**.
**Why (no hard SIZE-block):** A hard size gate on phase *advance* fails the anti-vanity litmus — cheapest pass is partition-gaming (split into sub-N commits), author-controlled, foreclosed by nothing; size correlates weakly with review-worthiness (a 30-line auth change outranks a 400-line fixture). So size never *blocks the phase from closing*. What size DOES legitimately ladder is whether the cross-model review is recommended vs runs: it is risk-correlated-enough via the drift accumulation to decide *urgency*, not *permission-to-advance*.
**Escalation rule (RESOLVED — user call):** when Layer 2 fires **RED**, the codex review is **MANDATORY and auto-runs** (no y/N skip) — the cross-model second opinion is never silently skipped when change has genuinely accumulated. **Degraded-confirm fallback (flexibility for users without codex):** the codex bridge is attempted; if the codex binary is absent or the run errors / returns empty (the documented macOS 127 / 0-byte / timeout case — see [[feedback-codex-bridge-timeout-macos]]), the skill surfaces the failure and requires an **explicit user confirmation** to proceed without the cross-model review, recording `Codex review: SKIPPED (codex unavailable, user-confirmed) at <ISO>` verbatim and surfacing it loudly at archive. Rationale for auto-run over prompt-y/N: codex is flat-rate (subscription) here, so per-run cost is a non-issue, and auto-run forecloses the "skipped by forgetting" failure the falsifier targets; the only legitimate skip is codex-genuinely-unavailable, which the degraded path captures explicitly rather than silently. Yellow stays advisory (surfaced, operator may run or not).
**Falsifier:** Auto-run-on-red fires so often on benign large-but-mechanical diffs that its findings get reflexively dismissed (rubber-stamp → red predicate is mis-tuned, too broad); OR the degraded-confirm "codex unavailable" path becomes a routine silent escape (users who never install codex confirm-skip every red, so the mandatory review is mandatory in name only — then the lane isn't really cross-model-gated for them); OR the scope-drift trigger never fires in practice (the one new risk signal is inert).
**Binds:** gate work after the Inc1 hard set (project-transitions.md phase-done + project-drift.md). Not on the critical path. Scope-drift trigger depends on the already-shipped `detect-scope`.

- **validate-state.js** — NO change: `main()` already accepts path args via `process.argv.slice(2)` → `collectTargets` (:251, 113-138); the `.atomic-skills` literal is only the no-arg fallback (:253).

Mechanism: B0 does `cp -a .atomic-skills .atomic-skills-dogfood`; every B1-D6 invocation passes the dogfood root. Live `.atomic-skills/` byte-frozen until D7. Tar snapshot kept **only** as the D7 belt-and-suspenders net, not as the isolation mechanism.
**Why:** Once (a) is near-free, it dominates on the merits that matter for a gitignored, non-git-restorable tree: (i) **diff-based bug attribution** — the byte-frozen original lets a corruption be `diff`ed to localize the offending tool/step (tar-only and copy-at-default both destroy the baseline); (ii) **zero-cost continuous rollback** — `rm -rf` the copy + re-`cp`; (iii) **coexistence guarantee** — dogfood writes only into the redirected root, so the live flat readers provably never see the half-migrated tree (the "half-migrated containment" invariant). Copy-at-default-path (b) is actively dangerous (a stray no-arg `validate-state` defaults to `.atomic-skills` and would hit the experiment); tar-only (c) forfeits attribution + coexistence for the whole multi-day window.
**Falsifier:** `materializeDecomposition` turns out to re-derive `.atomic-skills/` independently of opts (it doesn't — :624 docstring + :747-819 confirm it only returns `{relativePath, content}`); or running `node src/normalize.js .atomic-skills-dogfood` / `node scripts/validate-state.js .atomic-skills-dogfood` against the copy changes any live `.atomic-skills/` file's mtime (the redirect leaked).
**Binds:** Inc0 (establishes the convention) + the one code change folds into Inc2. Blocks the B0→D6 dogfood window (cheaply). Cross-deps: R-XAGENT-08 (same file), R-MIG-04/05 (same decompose edit), R-MIG-21 (`status/` rides in the `cp -a`; serve host-global paths stay un-redirected).
