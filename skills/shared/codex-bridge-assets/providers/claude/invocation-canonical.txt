Use this exact command shape for every Claude external-review invocation.
Departure from this shape risks interactive hangs, write-capable tools, intent
framing (CLAUDE.md), or stdout contamination.

## Variables to substitute

- `<BRIEFING_PATH>`: path to briefing markdown file (input via stdin redirect)
- `<OUTPUT_PATH>`: path to output markdown file (stdout redirect; no Codex-style `-o`)
- `<STDERR_PATH>`: path to stderr log (must live under a private `mktemp -d` dir)
- `<TIMEOUT_SECONDS>`: integer seconds (default 600 = 10 min)
- `<MODEL_FLAG>` / `REVIEW_MODEL_ID`: from Step 0.model. Empty when
  `source: cli-default` or `invalidModelId`. Prefer binding safe `REVIEW_MODEL_ID`
  and expanding `${REVIEW_MODEL_ID:+--model "$REVIEW_MODEL_ID"}` (quoted). Never
  invent an id; never pass an id that failed `isSafeReviewModelId`.
  Claude has **no live model catalog** ‚Äî aliases only (`opus`, `sonnet`, `haiku`,
  `fable`, ‚Ä¶) or full ids from the operator.

## Pre-step: private work directory (symlink-safe)

Never write stderr/output to a predictable shared `/tmp/‚Ä¶-<ts>` path. Create a
private dir first:

```bash
umask 077
REVIEW_TMP=$(mktemp -d "${TMPDIR:-/tmp}/as-claude-review.XXXXXX") || exit 1
trap 'rm -rf "$REVIEW_TMP"' EXIT
# Bind: <OUTPUT_PATH>=$REVIEW_TMP/out.md  <STDERR_PATH>=$REVIEW_TMP/stderr.log
#        <BRIEFING_PATH> may stay outside if already sealed by the caller
```

## Pre-step: portable timeout wrapper

`timeout(1)` is GNU coreutils. On macOS it is NOT installed by default; on
Linux + WSL it is. The skill MUST detect the available wrapper before
constructing the command ‚Äî never hardcode `timeout`.

```bash
if command -v timeout >/dev/null 2>&1; then
  run_with_timeout() { timeout "$@"; }
elif command -v gtimeout >/dev/null 2>&1; then
  run_with_timeout() { gtimeout "$@"; }
elif command -v perl >/dev/null 2>&1; then
  run_with_timeout() { perl -e 'alarm shift @ARGV; exec @ARGV' -- "$@"; }
else
  echo "ERROR: no timeout wrapper available. Install coreutils or perl." >&2
  exit 1
fi
```

All three branches expose the same shell function:
`run_with_timeout <SECONDS> <command...>`.

## Command (default ‚Äî OAuth-compatible)

```bash
# REVIEW_MODEL_ID must be empty or pass isSafeReviewModelId (slug-shaped).
# Briefing channel (proven): stdin redirect into -p
run_with_timeout <TIMEOUT_SECONDS> claude \
  --safe-mode \
  -p \
  ${REVIEW_MODEL_ID:+--model "$REVIEW_MODEL_ID"} \
  --effort xhigh \
  --tools "Read,Grep,Glob" \
  --permission-mode dontAsk \
  --no-session-persistence \
  --disable-slash-commands \
  --output-format text \
  < <BRIEFING_PATH> \
  > <OUTPUT_PATH> 2> <STDERR_PATH>
```

## Optional bare mode (API key only)

Only when `ATOMIC_SKILLS_CLAUDE_BARE=1` **and** `ANTHROPIC_API_KEY` is set.
`--bare` skips OAuth/keychain and CLAUDE.md/hooks (faster CI). Do **not** default
to bare ‚Äî most operators use `claude auth login`.

```bash
run_with_timeout <TIMEOUT_SECONDS> claude \
  --bare \
  -p \
  ${REVIEW_MODEL_ID:+--model "$REVIEW_MODEL_ID"} \
  --effort xhigh \
  --tools "Read,Grep,Glob" \
  --permission-mode dontAsk \
  --no-session-persistence \
  --disable-slash-commands \
  --output-format text \
  < <BRIEFING_PATH> \
  > <OUTPUT_PATH> 2> <STDERR_PATH>
```

## Flag-by-flag rationale

| Flag | Why |
|------|-----|
| `run_with_timeout <N>` | External kill. Portable across Linux/macOS/WSL. |
| `--safe-mode` | Strip CLAUDE.md / skills / plugins / hooks / memory (anti-framing) while keeping OAuth. |
| `-p` / `--print` | Headless one-shot; response on stdout. |
| `--tools "Read,Grep,Glob"` | Read-only allowlist. Reviewer must never write. |
| `--permission-mode dontAsk` | Sealed non-interactive surface. **Never** `bypassPermissions` or `--dangerously-skip-permissions` for review. |
| `--no-session-persistence` | Ephemeral; do not resume between Pass 1 and Pass 2. |
| `--disable-slash-commands` | Skills must not inject framing. |
| `--output-format text` | Markdown-friendly final text on stdout. |
| `--effort xhigh` | Always xhigh for sealed external review (deeper reasoning than high; default product choice ‚Äî not max, which has diminishing returns / overthinking). |
| `< <BRIEFING_PATH>` | Prompt from stdin (proven 2.1.212 dogfood). |
| `> <OUTPUT_PATH>` | Persist response for Pass validation. |
| `2> <STDERR_PATH>` | Keep auth/banner out of the review file. |

## DO NOT

- Use `--dangerously-skip-permissions` or `--permission-mode bypassPermissions` for review.
- Use `--tools` that include Edit, Write, or unrestricted Bash for review.
- Resume sessions (`-c` / `--resume`) between Pass 1 and Pass 2 ‚Äî each pass is a **fresh** headless run.
- Hardcode `timeout` (fails on macOS without coreutils). Use the detector above.
- Treat same-family host=Claude + `--mode=claude` as cross-model: route via
  `host-default-external.md` (confirm‚Üílocal or HARD ABORT).
- Claim a live model catalog exists (`claude models` is **not** a catalog command).

## Exit codes and failure signals

- `0` with non-empty `<OUTPUT_PATH>`: ok, parse the output file.
- `124`: timeout (`timeout` / `gtimeout`).
- `142` or SIGALRM-killed: timeout (perl `alarm` fallback).
- Non-zero **or** stderr containing auth failure markers: abort with preflight auth message.
- other non-zero: Claude error. Abort with message + point at the stderr log.

Both timeout exit codes (124 and 142) MUST be handled as "timeout" by the
skill ‚Äî the wrapper choice is invisible to the user.
