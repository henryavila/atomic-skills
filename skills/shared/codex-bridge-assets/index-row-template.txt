# Reviews INDEX.md Row Template

Path: `.atomic-skills/reviews/INDEX.md`

## File header (write only if INDEX.md does not exist)

```markdown
# Reviews Index

| Date | Topic | Skill | Provider | Verdict | Counts (final) | Framing Δ |
|------|-------|-------|----------|---------|----------------|-----------|
```

## Row to append per review

```markdown
| {{DATE_HHMM}} | [{{SLUG}}]({{FILENAME}}) | {{SKILL_SHORT}} | {{PROVIDER}} | {{VERDICT}} | {{COUNTS_COMPACT}} | {{DELTA_COMPACT}} |
```

## Placeholder formats

| Placeholder | Format |
|-------------|--------|
| `{{DATE_HHMM}}` | `YYYY-MM-DD HH:MM` |
| `{{SLUG}}` | kebab-case slug |
| `{{FILENAME}}` | `YYYY-MM-DD-HHMM-<slug>.md` (relative link from INDEX.md) |
| `{{SKILL_SHORT}}` | `plan` or `code` |
| `{{PROVIDER}}` | `codex` \| `grok` \| `local` (from review frontmatter; legacy rows without Provider column: treat as `codex` when reading only) |
| `{{VERDICT}}` | from review frontmatter |
| `{{COUNTS_COMPACT}}` | `<B>B/<C>C/<M>M/<m>m/<n>n` e.g. `1B/1C/3M/1m/0n` |
| `{{DELTA_COMPACT}}` | `<d>d/<=>=/<+>+` e.g. `2d/4=/1+` (dropped/maintained/emerged) |

### Migration note (old INDEX rows)

Pre-F3 INDEX headers lack the **Provider** column (6 columns). When appending
new rows, rewrite the header to the 7-column form above if missing Provider —
and in the **same atomic rewrite**, insert `codex` as the Provider cell into
every legacy 6-column data row (Date | Topic | Skill | **codex** | Verdict |
Counts | Framing Δ). Do not leave a mixed 6/7-column table. When *reading*
without rewriting, assume `provider: codex` for a 6-column legacy row.
